# Cypher Batch Fix - Issues #4184, #4185, #4186, #4188, #4189

**Branch:** fix/4184-4185-4186-4188-4189-cypher-batch
**Status:** done

## Issues fixed (all 5 on a single branch)

### #4186 - `reduce()` over null list

`ReduceExpression.evaluate()` returned the initial accumulator when the list operand was null. Per Cypher semantics, a null list operand makes the whole `reduce()` expression null. One-line fix in `ReduceExpression.java` and two new tests in `CypherReduceAndShortestPathTest`.

### #4184 - Map projection with function-call computed field

`CypherExpressionBuilder.findFunctionInvocationRecursive()` was descending into `MapProjectionContext` and surfacing the inner function call as the top-level expression, replacing the surrounding projection. The function already skipped `MapContext` for the same reason; the fix adds the parallel skip for `MapProjectionContext`. New tests in `OpenCypherExpressionTest` cover `toUpper(p.name)` and `size(friends)` computed fields.

### #4189 - `RETURN DISTINCT *` collapses rows

`ProjectReturnStep` built its DISTINCT hash key from the unexpanded `*` return item, so every row produced the same key. Fix: when `isReturnAll()`, build the key from the sorted set of properties on the projected row instead. New regression test class `Issue4189ReturnDistinctStarTest` exercises distinct-star with two distinct people, with a trivial `WITH`, with non-distinct duplicates, and the no-DISTINCT control.

### #4188 - Variable-bound endpoint property map

`MatchRelationshipStep.matchesTargetProperties()` accepted only the target vertex; it could not resolve an `Expression`-typed entry like `{name: fName}`. `MatchNodeStep` already handled this. Fix: pass `lastResult` through and evaluate `Expression` entries against it. New regression test class `Issue4188EndpointVariablePropertyTest` covers WITH and UNWIND sources plus the WHERE/literal control cases.

### #4185 - FOREACH DELETE not visible to later MATCH

Two issues compounded here:

1. `DeleteClause` only stored each target as the original expression text. `endNode(r)` was passed to a chained-access resolver that does not understand function calls, so the deletion was a silent no-op. Fix: also store the parsed `Expression` list and evaluate it for function-like targets in `DeleteStep`.
2. Once `endNode(r)` actually resolved, the Cypher pattern `DELETE endNode(r) DELETE r` raised `DeleteConnectedNode` because ArcadeDB deletes immediately while Cypher defers deletion to end of statement. Fix: in `ForeachStep`, deferred-delete batching per iteration. `DeleteStep` appends targets to a `CommandContext` variable when set; `ForeachStep` flushes them with edges-first ordering at the end of each iteration.

New regression test class `Issue4185ForeachDeleteVisibilityTest`.
